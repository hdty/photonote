# PhotoNote 仕様書（SPEC）

## 背景・目的
後期高齢者の父親が、デジカメで撮った写真を印刷するのに簡単なソフトが無く、
Windows7 の PC を使い続けている。Windows11 環境へ移行させたい。
そこで Windows 10 以上で動く、シンプルな写真印刷ソフトを自作する。

最重要は「高齢者が迷わず使えること」。機能の豊富さより、操作の簡単さを優先する。

## ターゲット環境
- 対象 OS: Windows 10 以上（64bit）
- 実装: C# / .NET 8 / WPF
- 主たる利用者: 高齢者（PC操作に不慣れ）
- 配布: .NET ランタイム同梱の単一ファイル exe（インストール不要）

## 名称・ブランド
- プログラム名: PhotoNote
- サークル: HidéToys（ヒデトイズ）名義で公開。
- 公開先: https://github.com/hdty/photonote （MIT、画像は対象外）。

## 機能要件

### 1. 編集画面（写真選び）  ※ sample/photonotesample02.pdf 参照
- 指定したフォルダ内の写真を一覧表示する。
- 一覧は **2列×4行（1画面8枚）** のページ送り式。「前へ / 次へ」で移動する
  （スクロールではなくページ切替の方が高齢者に分かりやすいため）。
- 各写真は「ファイル名 / キャプション入力欄 / 撮影日」の3段で表示する。
- 並び順を選べる: 撮影日時・ファイル名・更新日時 × 昇順・降順（既定は撮影日時の古い順）。
  日時は時刻まで含めて比較する。並び順はフォルダごとに保存し、印刷もこの順を保つ。
- 印刷したい写真をチェックボックスで複数選択できる（「すべて選択 / 選択を外す」あり）。

### 2. 印刷画面（設定 → プレビュー → 印刷）  ※ sample/photonotesample01.pdf 参照
- 編集画面と同一ウィンドウ内で切り替える（別ウィンドウにしない）。
  「写真選びに戻る」で戻れる。
- 選べる設定:
  - 用紙サイズ: A4 / B5 / 2L判 / L判 / はがき
  - 用紙の向き: 縦 / 横
  - 1ページの写真枚数: 1 / 2 / 4
  - 写真の並べ方: 縦 / 横（2枚のときのみ有効）
- 右側に印刷プレビュー（DocumentViewer）を表示。既定は「ページ全体を表示」。
- 「印刷する」でプリンタドライバの印刷ダイアログを開く
  （画質・部数・用紙種類はドライバ側で設定させるため。アプリ独自のプリンタ選択はしない）。
- 各写真の下に、左寄せでキャプション、右寄せで撮影日を出力する。
  - 撮影日は写真の EXIF（DateTaken）から取得。取れなければファイルの更新日時で代用。
  - 表記は `2025/5/7` 形式。

#### 印刷レイアウトの確定仕様
- 紙端の余白: B5以上（A4・B5）= 上下左右 10mm、B5未満（2L・L判・はがき）= 5mm。
  判定は用紙の短辺が 182mm（B5短辺）以上かどうか。
  （フチあり印刷の必要余白の実測: EPSON EW-M873T=四辺3mm、Canon PIXUS TS8630=左右5mm/上下3.4mm）
- キャプションのフォント: Meiryo UI。
- 文字サイズ（固定）: B5以上 = 10pt、B5未満 = 8pt。1ページの枚数では変えない。
- 写真とキャプションの間隔: 文字サイズの約2割。

### 3. データ保存（フォルダ単位の隠しファイル）
- キャプション・選択状態・並び順・印刷設定を、写真フォルダごとに固定ファイル名の隠しファイルへ保存する。
- ファイル名は `.photonote.json`（固定）。フォルダ内に1つ。Windows の隠し属性を付与。
- 保存タイミング: 入力が止まって1.5秒後、フォルダ切替時、終了時。

## 隠しファイル形式（.photonote.json）
※ 実装（`PhotoNote/NoteFile.cs`）が正。以下は実出力に基づく。食い違ったら実装に合わせて更新する。

- 配置: 各写真フォルダの直下に1つ。
- ファイル名: `.photonote.json`（固定）。Windows の隠し属性を付与。
- 文字コード: UTF-8（BOMなし）。
- 実際の形式:

```json
{
  "app": "PhotoNote",
  "version": 1,
  "updatedAt": "2026-06-11T17:13:04+09:00",
  "sortOrder": "DateTakenAsc",
  "printSettings": {
    "paperSize": "A4",
    "orientation": "Portrait",
    "photosPerPage": 2,
    "layout": "Vertical"
  },
  "photos": [
    { "file": "IMG_0001.jpg", "text": "入場シーン", "selected": true },
    { "file": "IMG_0002.jpg", "text": "",           "selected": false }
  ]
}
```

- `app` / `version`: 識別子とフォーマット版。
- `updatedAt`: 最終保存日時（ISO 8601、ローカルタイムゾーン付き）。
- `sortOrder`: `DateTakenAsc`（既定）/ `DateTakenDesc` / `FileNameAsc` / `FileNameDesc` /
  `ModifiedAsc` / `ModifiedDesc`。いずれも時刻まで比較。
- `printSettings`: 直近の印刷設定（次回の既定値として復元）。
  - `paperSize`: `A4` / `B5` / `2L` / `L` / `Hagaki`
  - `orientation`: `Portrait` / `Landscape`
  - `photosPerPage`: `1` / `2` / `4`
  - `layout`: `Vertical` / `Horizontal`（2枚時の並べ方）
- `photos[]`: ファイル名ごとのキャプションと選択状態。
  - `file`: フォルダからの相対ファイル名。
  - `text`: キャプション（未入力は空文字）。
  - `selected`: 印刷対象チェックの状態。
  - 配列の順序は現在の並び順（`sortOrder`）に従う。
  - フォルダから消えた写真でも、キャプションがあれば項目を残す。

### 撮影日について（設計判断: 確定済み）
- 撮影日は **JSON に保存しない**。表示・印刷のたびに EXIF（無ければ更新日時）から読む。
  → ファイルをキャプション・設定だけの最小構成に保つ方針で確定。

## アーキテクチャ（実装と一致）
- `ImageLoader.cs`         : 画像の読み込み（EXIF回転の反映、撮影日時の取得）。
- `PhotoItem.cs`           : 写真1枚のモデル（ファイル名・更新日時・撮影日・キャプション・選択・サムネイル）。
- `NoteFile.cs`            : `.photonote.json` の読み書き（隠し属性の付け外しを含む）。
  `PrintSettings` / `PhotoEntry` もここに定義。
- `PrintDocumentBuilder.cs`: 用紙・枚数・レイアウトに応じた印刷ドキュメント（FixedDocument）生成。
- `MainWindow.xaml(.cs)`   : 1ウィンドウ2画面（編集画面 / 印刷画面）の UI。
  開発用に `--uishot <写真フォルダ> <出力フォルダ>` で両画面をPNG保存する隠しオプションあり。

## 補助ツール（tools/）
- `IconTool/`                  : photonoteicon01.png から角丸グラデーション背景付き .ico を生成。
- `Set-JapaneseVersionLang.ps1`: 発行済み exe のバージョン情報の言語をニュートラル→日本語に書き換える
  （.NET の仕様で言語がニュートラル固定になるため、リリース時に実行する）。

## 対象外 / 将来検討
- 写真編集（トリミング・色補正等）は対象外（印刷とキャプションに専念）。
- ヘルプ画面（HidéToysロゴはここで使う想定。現在ロゴは埋め込み済みだが通常画面では未使用）。
- コード署名（未署名のため初回起動時に SmartScreen 警告が出る。費用面から当面は見送り）。
